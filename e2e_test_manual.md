# 端到端测试脚本

## 测试目标
1. 顶部mode tabs不可点击
2. 重新生成按钮正确工作（所有模式）
3. LaTeX渲染无残留

## 测试步骤

### 1. 顶部Tabs测试
- [ ] 访问 http://127.0.0.1:8080/ui/
- [ ] 选择任一模式进入chat视图
- [ ] 尝试点击顶部mode tabs，应无反应
- [ ] 鼠标悬停在tabs上，cursor应为default而非pointer
- [ ] 只能通过点击"返回主界面"按钮切换模式

### 2. 重新生成按钮测试

#### 2.1 学习模式
- [ ] 输入一个数学命题（如："证明勾股定理"）
- [ ] 选择level（如"本科"）
- [ ] 等待生成完成
- [ ] 点击重新生成按钮
- [ ] 验证：不应重复显示用户消息，level参数应保持

#### 2.2 问题求解模式
- [ ] 输入一个问题
- [ ] 等待求解完成
- [ ] 点击重新生成按钮
- [ ] 验证：不应重复显示用户消息

#### 2.3 证明审查模式（文本）
- [ ] 粘贴一段证明文本
- [ ] 等待审查完成
- [ ] 点击重新生成按钮
- [ ] 验证：不应重复显示用户消息，文本应保留

#### 2.4 证明审查模式（PDF）
- [ ] 上传PDF文件
- [ ] 等待审查完成
- [ ] 点击重新生成按钮
- [ ] 应提示"PDF附件无法自动恢复，请重新上传"

#### 2.5 定理检索模式
- [ ] 输入检索查询
- [ ] 等待检索完成
- [ ] 点击重新生成按钮
- [ ] 验证：不应重复显示用户消息

### 3. LaTeX渲染测试

#### 3.1 证明审查（文本）
- [ ] 提交包含LaTeX的证明文本（如：\\textbf{Theorem} Let $x^2$ be...）
- [ ] 检查输出中是否有`\textbf`, `\cite`, `\ref`等残留
- [ ] 数学公式`$x^2$`应正确渲染

#### 3.2 证明审查（PDF）
- [ ] 上传包含LaTeX的PDF
- [ ] 检查章节审查结果中是否有LaTeX控制序列残留
- [ ] 数学公式应正确渲染

#### 3.3 其他模式
- [ ] 学习模式：检查输出中无LaTeX残留
- [ ] 问题求解：检查证明blueprint中无LaTeX残留
- [ ] 定理检索：检查定理声明中无LaTeX残留

## 预期结果
- 所有测试通过
- 无JavaScript错误
- UI交互符合预期
