═══════════════════════════════════════════════════════════════════════════════
                              数据流与通信协议
═══════════════════════════════════════════════════════════════════════════════

用户操作
   ↓
前端UI (JavaScript)
   ↓
【同步请求】
   POST /config/llm        → 保存配置到config.toml
   GET  /health            → 获取系统状态
   
【异步流式请求 (SSE)】
   POST /learn + SSE       → 流式返回Learning内容
   POST /solve + SSE       → 流式返回解题过程
   POST /review + SSE      → 流式返回审阅结果
   
   SSE消息格式:
   • <!--vp-status:...-->  状态更新
   • <!--vp-result:...-->  部分结果
   • <!--vp-final:...-->   最终结果
   
   ↓
后端处理 (FastAPI)
   ↓
调用LLM API (DeepSeek等)
   ↓
流式返回给前端
   ↓
前端渲染 (KaTeX + Marked.js)
   ↓
用户看到结果

## 架构特点总结

### 优势 ✓
1. **简单直接** - 无复杂构建流程，开发效率高
2. **模块化** - 前后端清晰分离，各模式独立
3. **可扩展** - 易于添加新模式和功能
4. **用户友好** - 流式响应，实时反馈
5. **配置灵活** - 支持多个LLM提供商

### 技术亮点 ✨
1. **SSE流式传输** - 更好的用户体验
2. **LaTeX渲染** - 完美的数学公式显示
3. **结构化输出** - Learning模式的4段式讲解
4. **持久化配置** - config.toml可靠存储
5. **国际化支持** - 中英文无缝切换
